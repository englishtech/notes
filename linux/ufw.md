# 🛡️ UFW: Краткая шпаргалка

**UFW (Uncomplicated Firewall)** — интерфейс для управления сетевым фильтром в Linux.
**Зачем:** Блокирует все входящие соединения по умолчанию. Открывает только явно указанные порты. Защищает от сканирования и несанкционированного доступа.

---

## ⚡️ Быстрый старт

⚠️ **Сначала разреши SSH, иначе потеряешь доступ!**

1.  **Разрешить SSH:**
    ```bash
    sudo ufw allow 22/tcp
    ```
2.  **Включить фаервол:**
    ```bash
    sudo ufw enable
    ```
3.  **Проверить статус:**
    ```bash
    sudo ufw status
    ```

---

## 🔌 Основные порты

| Порт | Протокол | Назначение | Команда |
| :--- | :--- | :--- | :--- |
| **22** | TCP | SSH (удаленное управление) | `sudo ufw allow 22/tcp` |
| **80** | TCP | HTTP (веб-сайты) | `sudo ufw allow 80/tcp` |
| **443** | TCP | HTTPS (защищенный веб) | `sudo ufw allow 443/tcp` |
| **25** | TCP | SMTP (почта) | `sudo ufw allow 25/tcp` |
| **53** | UDP/TCP | DNS | `sudo ufw allow 53` |

*Исходящий трафик (запросы от сервера наружу) разрешен по умолчанию.*

---

## 🛠 Базовые команды

| Действие | Команда |
| :--- | :--- |
| **Посмотреть статус** | `sudo ufw status` |
| **Посмотреть с номерами** | `sudo ufw status numbered` |
| **Открыть порт** | `sudo ufw allow <порт>/tcp` |
| **Закрыть порт** | `sudo ufw deny <порт>/tcp` |
| **Удалить правило** | `sudo ufw delete allow <порт>/tcp` |
| **Выключить** | `sudo ufw disable` |
| **Сбросить всё** | `sudo ufw reset` |
| **Включить логи** | `sudo ufw logging on` |

---

## ⚠️ Важно
*   Если закрыл порт 22 случайно: используй консоль хостинга (VNC/Web Console) для доступа, затем `sudo ufw disable`.
*   Приоритет правил: конкретные IP важнее общих портов (`allow from 1.2.3.4 to any port 22`).
