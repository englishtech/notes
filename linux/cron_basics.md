# Cron

**Cron** — это демон планировщика задач в Unix-подобных системах. Он позволяет автоматически выполнять команды или скрипты в заданное время.

## 1. Основные команды

| Команда | Описание |
| :--- | :--- |
| `crontab -e` | Редактировать расписание текущего пользователя |
| `crontab -l` | Показать текущее расписание |
| `crontab -r` | Удалить всё расписание (осторожно!) |
| `sudo systemctl status cron` | Проверить, запущен ли сервис cron |

## 2. Синтаксис Crontab

Каждая строка состоит из **5 полей времени** и **команды**:

```text
* * * * * команда_для_выполнения
│ │ │ │ │
│ │ │ │ └── День недели (0–7, где 0 и 7 — Воскресенье)
│ │ │ └──── Месяц (1–12)
│ │ └────── День месяца (1–31)
│ └──────── Час (0–23)
└────────── Минута (0–59)
```

### Примеры периодичности

| Значение | Описание | Пример |
| :--- | :--- | :--- |
| `* * * * *` | Каждую минуту | `* * * * * /usr/bin/date >> /tmp/date.log` |
| `*/15 * * * *` | Каждые 15 минут | `*/15 * * * * python3 script.py` |
| `0 * * * *` | Каждый час (в 00 минут) | `0 * * * * backup.sh` |
| `0 0 * * *` | Каждый день в полночь | `0 0 * * * certbot renew` |
| `0 9 * * 1` | Каждый понедельник в 9:00 | `0 9 * * 1 report.py` |
| `@reboot` | При загрузке системы | `@reboot /home/user/start_bot.sh` |

## 3. Важные правила

1.  **Полные пути:** Cron не знает о ваших переменных окружения (`PATH`). Всегда указывайте полные пути к интерпретаторам и файлам.
    *   ❌ `python3 main.py`
    *   ✅ `/home/user/project/.venv/bin/python3 /home/user/project/main.py`
2.  **Рабочая директория:** Cron запускает задачи из домашней директории пользователя. Если скрипт зависит от относительных путей, используйте `cd`:
    *   `cd /home/user/project && ./script.sh`
3.  **Логирование:** По умолчанию вывод никуда не идет. Перенаправляйте stdout и stderr в файл:
    *   `>> /path/to/log.log 2>&1`

## 4. Работа с логами

Чтобы понять, почему скрипт не сработал или сработал с ошибкой:

```bash
# Посмотреть последние 50 строк лога
tail -n 50 /home/user/project/cron.log

# Следить за логом в реальном времени
tail -f /home/user/project/cron.log
```

Если вы не настроили перенаправление в файл, системные логи cron можно найти здесь:
```bash
grep CRON /var/log/syslog
# или
journalctl -u cron -f
```

# Быстрые примеры

**Синтаксис:** `минута час день месяц день_недели команда`

### 1. Стандартный запуск Python
Запуск скрипта из виртуального окружения с записью логов.
```cron
0 9 * * * /home/user/project/.venv/bin/python3 /home/user/project/main.py >> /home/user/project/log.txt 2>&1
```

Если скрипт зависит от файлов в своей папке:
```cron
*/15 * * * * cd /home/user/project && .venv/bin/python3 main.py >> log.txt 2>&1
```
