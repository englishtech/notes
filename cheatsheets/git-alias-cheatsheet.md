# Git Alias: Быстрый коммит и пуш одной командой

Алиас `git gp`, который заменяет три команды (`add`, `commit`, `push`) на одну короткую фразу:
```bash
git config --global alias.gp '!f() { git add . && git commit -m \"$*\" && git push; }; f'
```

В любой папке с git-репозиторием:
```bash
git gp "Сообщение коммита"
```

---

Команда создает запись в файле `~/.gitconfig`. Вот подробный разбор синтаксиса:

| Часть кода | Описание |
| :--- | :--- |
| `git config --global` | Сохраняет настройку глобально для всех проектов пользователя. |
| `alias.gp` | Имя нового алиаса. Теперь можно писать `git gp`. |
| `'!...'` | Восклицательный знак говорит Git: «Запусти это как **bash-скрипт**, а не как встроенную команду git». |
| `f() { ... }; f` | Объявление и мгновенный запуск временной функции `f`. Это нужно, чтобы корректно передавать аргументы внутрь скрипта. |
| `git add .` | Добавляет все измененные файлы в индекс. |
| `&&` | Логическое «И». Следующая команда выполнится **только если** предыдущая прошла успешно (код возврата 0). |
| `git commit -m "$*"` | Делает коммит. <br> **`$*`** — подставляет все аргументы, введенные после `git gp`, в одну строку сообщения. |
| `git push` | Отправляет изменения на удаленный сервер. |

### Логика безопасности
Благодаря операторам `&&`, цепочка прервется при первой же ошибке:
*   Если нет изменений для `add` → коммит не создастся.
*   Если коммит провалился (например, конфликт) → `push` не выполнится.

---

Настройка сохраняется в файле конфигурации Git:
*   **Linux/macOS:** `~/.gitconfig`
*   **Windows:** `C:\Users\<ТвоеИмя>\.gitconfig`

Можно проверить содержимое вручную:
```ini
[alias]
    gp = "!f() { git add . && git commit -m \"$*\" && git push; }; f"
```

Убрать эту команду:
```bash
git config --global --unset alias.gp
```
