# 🌐 Сетевая шпаргалка

> Кратко и просто о сетевых технологиях.

---

## 📍 Сетевые адреса

| Термин | Что это | Зачем | Пример |
|--------|---------|-------|--------|
| **IP-адрес** | Уникальный адрес устройства в сети | Чтобы находить устройства и передавать данные | `192.168.1.10`, `8.8.8.8` |
| **MAC-адрес** | Физический адрес сетевой карты (заводской) | Доставка данных внутри одной локальной сети | `00:1A:2B:3C:4D:5E` |
| **DHCP** | Сервис авто-выдачи настроек сети | Чтобы не прописывать IP, маску, шлюз вручную | Подключился к Wi-Fi → получил IP автоматически |
| **Маска подсети** | Фильтр: какая часть IP = сеть, какая = устройство | Понять: адрес получателя «свой» или «чужой» | `255.255.255.0` или `/24` |
| **NAT** | Подмена внутреннего адреса на внешний | Выход многих устройств в интернет через один публичный IP | Домашний роутер: все гаджеты → один «белый» IP |

---

## 🔄 Маршрутизатор и коммутатор

| Устройство | Уровень | Зачем | Пример |
|------------|---------|-------|--------|
| **Router** | Сетевой (L3) | Соединяет разные сети, маршрутизирует трафик | Домашний роутер: соединяет вашу LAN с интернетом (WAN) |
| **Switch** | Канальный (L2) | Соединяет устройства внутри одной сети по MAC-адресам | Офисный коммутатор: 24 порта для ПК, принтеров, камер |

---

## 🌍 WAN и LAN

* **LAN** (Local Area Network) — локальная сеть: дом, офис, здание. Быстрая, своя. Пример: `192.168.1.0/24`.
* **WAN** (Wide Area Network) — глобальная сеть: интернет, сеть провайдера. Медленнее, чужая инфраструктура.

---

## 📛 DNS (Domain Name System)

* **Что**: «Телефонная книга интернета».
* **Зачем**: Преобразует понятные имена в IP-адреса.
* **Пример**: `google.com` → `142.250.185.46`. Без DNS пришлось бы запоминать цифры.

## 📦 Протоколы транспорта

| Протокол | Надёжность | Скорость | Когда использовать | Пример |
|----------|------------|----------|-------------------|--------|
| **TCP** | ✅ Гарантирует доставку, порядок, повтор при потере | Медленнее | Веб, почта, файлы — где важна целостность | Загрузка страницы, отправка письма |
| **UDP** | ❌ Отправил и забыл (нет гарантий) | Быстрее | Видео, голос, игры — где важна скорость, а не каждый пакет | Видеозвонок, онлайн-игра, стрим |

## 🔌 Порты

* **Что**: Числовой «вход» в устройстве для конкретного сервиса (0–65535).
* **Зачем**: Чтобы один IP мог обслуживать много задач одновременно.
* **Виды**:
  * **Well-known (0–1023)**: `80` (HTTP), `443` (HTTPS), `22` (SSH)
  * **Registered (1024–49151)**: `3306` (MySQL), `8080` (альт. веб)
  * **Dynamic (49152–65535)**: временные порты клиентов

## 🌐 HTML, HTTP, HTTPS

| Термин | Что это | Пример |
|--------|---------|--------|
| **HTML** | Язык разметки веб-страниц | `<h1>Заголовок</h1>` |
| **HTTP** | Протокол передачи веб-данных (без шифрования) | `GET /index.html` |
| **HTTPS** | HTTP + шифрование (SSL/TLS) | `https://bank.ru` — безопасно |

---

## 📡 HTTP-методы

| Метод | Зачем | Пример |
|-------|-------|--------|
| `GET` | Получить данные | Открыть страницу |
| `POST` | Отправить данные (создать) | Форма регистрации |
| `PUT` | Обновить данные целиком | Редактировать профиль |
| `DELETE` | Удалить данные | Удалить пост |
| `PATCH` | Частичное обновление | Сменить только аватар |

## 🔐 VPN и Прокси

| Технология | Как работает | Зачем | Пример |
|------------|--------------|-------|--------|
| **Прокси** | Посредник для одного приложения | Скрыть свой IP, обойти блокировку | Браузер → прокси → сайт |
| **VPN** | Зашифрованный туннель для всего трафика | Безопасность в публичных сетях, доступ к ресурсам | Телефон → VPN-сервер → интернет |

> ⚠️ Прокси не шифрует весь трафик, VPN — шифрует.

---

## 🔗 API (Application Programming Interface)

* **Что**: Набор правил, как одной программе общаться с другой.
* **Зачем**: Получать данные или функции внешнего сервиса без «взлома».
* **Пример**: Погодное приложение отправляет `GET /weather?city=Samara` → получает JSON с температурой.

## 📚 Модель OSI (7 уровней)

```
7. Прикладной  — приложения (браузер, почта)     → HTTP, DNS
6. Представления — кодирование, шифрование        → SSL, JPEG
5. Сеансовый   — управление сессией              → установление связи
4. Транспортный — доставка (TCP/UDP)             → порты, надёжность
3. Сетевой     — маршрутизация (IP)             → IP-адреса, пути
2. Канальный   — доставка в локалке (MAC)       → коммутаторы, ARP
1. Физический  — кабели, сигналы                → витая пара, Wi-Fi
```
