1\) Создать SSH-ключ на сервере (удаленном компьютере) (современный алгоритм `ed25519`):

```bash
ssh-keygen -t ed25519 -C "key-for-my-server"
```

На вопрос `Enter file in which to save the key` нажать **Enter** (оставить путь по умолчанию `/root/.ssh/id_ed25519`).  
На вопрос `Enter passphrase` нажать **Enter** дважды (оставить пустым).  
(Чтобы скрипты и автоматические обновления работали без ввода пароля).

2\) Скопировать публичный ключ - нужно вывести содержимое созданного файла, чтобы скопировать его.

```bash
cat ~/.ssh/id_ed25519.pub
```

Результат - длинная строка, начинающаяся с `ssh-ed25519 AAAA...` и заканчивающаяся `... key-for-my-server"`.  
Скопировать эту строку целиком.  

3\) Добавить ключ в аккаунт GitHub
- Зайти на [github.com](https://github.com) под своим логином.
- Кликнуть на аватарку (справа сверху) → **Settings**.
- В меню слева - **SSH and GPG keys**.
- Нажать зеленую кнопку **New SSH key**.
- Заполнить поля:
    *   **Title:** Любое название, например `key-for-my-server`.
    *   **Key type:** Оставить **Authentication Key**.
    *   **Key:** Вставить скопированную строку из Шага 2.
- Нажать **Add SSH key**.
    *   Если спросит пароль от GitHub — ввести его.

4\) Проверить подключение - в терминале (на VPS или локальном копьютере) выполнить тестовую команду:

```bash
ssh -T git@github.com
```

**Правильный ответ:**
```text
Hi твой-логин! You've successfully authenticated, but GitHub does not provide shell access.
```
*(Сообщение "does not provide shell access" — это норма, мы не хотим заходить в оболочку GitHub, нам нужен только доступ к репозиториям).*  
**Готово!** Теперь можно делать `git clone`, `git pull` без паролей.

---

Как использовать теперь?  
`git clone git@github.com:user/repo.git`

*(Чтобы получить правильную ссылку: на странице репозитория нажать зеленую кнопку **Code** → выбрать вкладку **SSH** → скопировать ссылку).*
